Hbases实践

发表于 2016-11-13 | 更新于 2018-11-22 | 分类于 大数据 | 评论数： | 阅读次数：

What

非关系型数据库

  • 高可靠性
  • 高性能
  • 面向列
  • 可伸缩
  • 实时读写
  • 分布式

对比

  • 关系型数据库的优点
    • 容易理解
    • 使用方便
    • 易于维护
  • 关系型数据库的瓶颈
    • 高并发读写需求
    • 海量数据的读写性能低
    • 扩展性和可用性差

数据库模型

  • row key
    • 不能重复
    • 字典顺序排序
    • 只能存储64k的字节
  • 列族
    • 小于等于3个
  • 列名
    • 以列族作为前缀
    • 可动态加入

  • celll单元格

    • 有版本
    • 字节数组

  • 时间戳

    • 默认是1
    • 时间倒序排序，最新最前
    • 64位整型
    • 默认精确到毫秒，可以主动设置
  • Hlog (wal log)
    • HLogkey 数据归属信息
      • table
      • region
      • sequence number
      • timestamp
    • value
      • 就是hbase的 keyvale对象

体系架构

  • client
  • zookeeper
    • 保证集群只有一个master
    • 存储 region的寻址入口
    • 监控region
  • master
    • 负责负责均衡
    • 发现失效region server 重新分配region
    • 管理用户CRUD
  • Region server
    • 维护region
  • resion
    -
  • store 一个store对应一个CF(列族)
    • memstore
    • storefile

高表与宽表的选择

  • 查询性能 高表更好
  • 分片能力 高表更细
  • 元数据开发 高表更大 rowkey 多，region 多 meta 数据量大
  • 事务能力 宽表事务性更好
  • 数据压缩比 宽表更高

How

squirrel sql client

  1. cd /opt/soft
  2. wget http://mirrors.cnnic.cn/apache/hbase/0.98.24/hbase-0.98.24-hadoop2-bin.tar.gz
  3. tar zxvf hbase-0.98.24-hadoop2-bin.tar.gz

  4. vi ~/.bash_profile
    增加

    发送打其它节点

  5. source ~/.bash_profile 生效

  6. vi /opt/soft/hbase-0.98.24-hadoop2/conf/hbase-env.sh
    修改

  7. vim /opt/soft/hbase-0.98.24-hadoop2/conf/hbase-site.xml

  8. vim /opt/soft/hbase-0.98.24-hadoop2/conf/regionservers

  9. vim /opt/soft/hbase-0.98.24-hadoop2/conf/backup-masters

  10. 拷贝hdfs的配置文件到hbase配置目录下
    cp /opt/soft/hadoop-2.5.1/etc/hadoop/hdfs-site.xml /opt/soft/hbase-0.98.24-hadoop2/conf/

  11. 将配置好的hbase发送到其它节点
    scp -r /opt/soft/hbase-0.98.24-hadoop2 root@sj-node2:/opt/soft/
    scp -r /opt/soft/hbase-0.98.24-hadoop2 root@sj-node3:/opt/soft/

  12. 在sj-node1节点上启动 start-hbase.sh

  13. 在webL浏览器访问 http://sj-node1:60010

  14. 试下ha高可用
    kill 调用sj-node1上的master
    自动切换完成


重新启动sj-node1节点 的hbase则变为backup

命令

  1. 打开命令行
    hbase shell
  2. help
  3. list
  4. create 'testhbase','cf1'
  5. put 'testhbase','rk00001','cf1:id','001'
  6. scan 'testhbase'
  7. get 'testhbase','rk00001'
  8. disable 'testhbase
  9. delete 'testhbase','rk00001','cf1:id'

    预分区

    在创建HBase表的时候默认一张表只有一个region，所有的put操作都会往这一个region中填充数据，当这个一个region过大时就会进行split。如果在创建HBase的时候就进行预分区则会减少当数据量猛增时由于region split带来的资源消耗。

  • create 'testsplit','cf1',SPLITS => ['000','100','200']
  • put 'testsplit','002','cf1:id','002d'

优化

  1. 表的设计
    • 预分区
    • row key
      • 定长 越小越好
      • 根据实际业务来
      • 散列性
    • 列族不能超过3个
    • setinmemory 大部分能提高效率
    • TTL 存储时长
    • compaction 合并默认值
      • majorcompaction =24
      • jetter = 0.2
  2. 写表操作
    • 多表同时写
    • htable参数设置
    • auto flush
    • write buffer
    • wal
    • 批量写
    • 多线程并发写 map reduce 写
  3. 读表操作
    • 多htable 并发读
    • scan 指定列族
    • 关闭result
    • 批量读
    • blockcache
