Hadoop实践之MapReduce

发表于 2016-10-15 | 更新于 2018-11-09 | 分类于 大数据 | 评论数： | 阅读次数：

What

分布式离线计算框架

  • 移动计算而不是移动数据
  • 一主多从架构
    • JobTracker
    • TaskTracker
  • NameNode
  • DateNode
  • ResourceManager
    • container
      • 一个container一个进程
  1. input split 对应一个 block 128M
  2. output 可能大于 一个block 128M
  3. map 到 reduce 方法之间 叫 shuffle
  4. map 在相应的datanode上执行，reduce找空闲的datanode执行，不一定在block上
  5. split后的大小不一定是一个block大小
  6. map的输出物是经过快速排序的






### partition
- hash(key) mod r
- 有几个reduce 就有几个partition

### shuffle
map task

Why

海量数据计算

Where

http://www.cnblogs.com/luogankun/p/4019303.html

How

  1. 配置文件yarn-site.xml

  2. 配置mapred-site.xml使用yarn来调度
    cp etc/hadoop/mapred-site.xml.template etc/hadoop/mapred-site.xml
    vi etc/hadoop/mapred-site.xml

  3. 将配置文件拷贝到其它机器上

  4. 由于配置了3，4节点为yarn的主节点，所以进入3或4执行命令
    ./sbin/start-yarn.sh

  5. 如果某个节点故障 kill掉之后 执行单独启动命令./sbin/yarn-daemon.sh start resourcemanager
  6. 检查
    http://sj-node3:8088/cluster
    能够访问代表主节点启动成功
    http://sj-node4:8088/cluster
    能够访问并自动跳转到node3代表热备机启动成功
